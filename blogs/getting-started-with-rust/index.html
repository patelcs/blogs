Getting Started with Rust

Starting again from scratch, this time with practice and small blogs to stay consistent.

Mar 30, 2026 Series: The Rust Book Topic: Getting Started

Why I'm Starting Over

Hey everyone! From today onward I'm starting Rust from scratch again. I had started earlier, stopped, and forgot a lot. This time I'm writing small blogs alongside my learning so I can pick up where I left off and actually retain the concepts.

The biggest mistake I made before was not practicing enough. Without practice, things fade and it takes time to get back into the zone. So now I'm committing to consistent practice and documentation.

Learning Plan

I'm learning from The Rust Book. It can feel a bit theoretical at times, but I know it will teach me a lot of new concepts.

Along the way, I'll solve coding problems I've already done in other languages as well as new ones. I want to stay consistent this time, so let's start and see.

The introduction in the book explains who the book is for and how to use it. Prior coding experience helps you move faster, and without it some parts can feel harder at first.

1. Installation

I'm on Ubuntu 22, so I followed the official Rust installer. One command and Rust was ready to go:

$ curl --proto '=https' --tlsv1.2 https://sh.rustup.rs -sSf | sh

Rust was installed successfully. You can open the docs locally (including The Rust Book) with:

$ rustup doc

To keep toolchain up to date:

$ rustup update

To remove Rust from the system:

$ rustup self uninstall

2. Hello World

Create a project directory, add a main.rs file, and use the classic hello world:

fn main() {
    println!("Hello, world!");
}

Compile and Run

Compile the file (Rust is compiled) and run the output binary:

$ rustc main.rs
$ ./main

You should see "Hello, world!" printed in the terminal.

3. Hello Cargo

Cargo is Rust's build system and package manager, similar to npm in Node or pip in Python.

For small programs you might not need dependencies, but as projects grow, Cargo makes it easy to manage packages and builds without compiling everything manually.

Create a Cargo Project

Create a new project using Cargo:

$ cargo new hello_cargo

This creates a hello_cargo directory with Cargo.toml, a .gitignore file, and src/main.rs.

Project Structure

Cargo expects your source files inside the src directory. The top-level directory is for README files, license information, configuration files, and other metadata.

Build and Run

Build the project and run the binary:

$ cargo build
$ ./target/debug/hello_cargo

One Step Run

Build and run in a single step:

$ cargo run

To verify compilation quickly without producing a binary:

$ cargo check

optimized production build :

$ cargo build --release
