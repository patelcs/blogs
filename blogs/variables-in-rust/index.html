1. Variables and Mutability

We can declare variables using the let keyword as shown in the code below.

The syntax for declaring variables is: let <var_name>: <var_type> = <var_val>. We'll talk about variable types in upcoming blogs.

As shown in variable b's definition, the var_type is optional and determined by the compiler only because the value is initialized in the same statement.

If declaration and initialization are split like variable x, then var_type is mandatory, and the value can be given later. But we cannot use x before initialization as shown.

Variables are by default immutable in Rust. When a variable is immutable, once a value is bound to a name, you can't change that value. As shown in the code, if you try to assign a value to b or x you can't do it.

fn main() {
    let a: i32 = 5;
    println!("The value of a is: {a}");

    let b = 10;
    // b = 12; // gives compile error
    println!("The value of b is: {b}");

    let x: i32;
    // println!("The value of x is: {x}");  // gives compile error
    x = 15;
    // x = 20; // gives compile error
    println!("The value of x is: {x}");
}

Below code shows how we can make variables mutable using the mut keyword in order to change the value that a variable holds.

fn main() {
    let mut x = 5;
    println!("The value of x is: {x}");
    x = 6;
    println!("The value of x is: {x}");
}

2. Constants

Like immutable variables, constants are not allowed to change, but there are a few differences between constants and variables.

First, we aren't allowed to use mut with constants. Constants aren't just immutable by default, they're always immutable. The var_type must be specified for constants.

Constants can be declared in any scope, including the global scope, which makes them useful for values that many parts of code need to know about.

The last difference is that constants may be set only to a constant expression, not the result of a value that could only be computed at runtime. This means constants must be known at compile time.

Constants are valid for the entire time a program runs, within the scope in which they were declared.

We can declare constants using the const keyword.

fn main() {
    // const ONE_HOURS_IN_SECONDS = 60 * 60 * 1; // error: must specify variable type

    const TWO_HOURS_IN_SECONDS: u32 = 60 * 60 * 2;
    println!("The value of TWO_HOURS_IN_SECONDS is: {TWO_HOURS_IN_SECONDS}");

    let sim = 60;
    // const THREE_HOURS_IN_SECONDS: u32 = sim * 60 * 3;  // not allowed
}

3. Shadowing

We can declare a new variable with the same name as a previous variable using shadowing, like the code in the snippet below.

fn main() {
    let x = 5;
    let mut y = 9;
    let x = x + 1;
    {
        let x = x * 2;
        y = y * 2;
        println!("The value of x in the inner scope is: {x}");
        println!("The value of y in the inner scope is: {y}");
    }
    println!("The value of x is: {x}");
    println!("The value of y is: {y}");

    let characters = "abcdef";
    println!("The value of characters is: {characters}");

    let characters = characters.len();
    println!("The value of characters is: {characters}");

    let mut characters = "klmno";
    println!("The value of characters is: {characters}");

    characters = "xyz";
    println!("The value of characters is: {characters}");

    let characters = characters.len();
    println!("The value of characters is: {characters}");
}

Using shadowing we can achieve the following things:

  • Change the value of an immutable variable like x in the code above. This is safe because we explicitly do so.
  • Change the type of a variable like characters, which is first a string then shadowed to characters.len() (a different type).
  • Change the mutability of a variable like characters: first immutable, then mutable, then immutable again.
  • Shadow a variable in an inner scope without affecting the outer scope variable. In the code above, the inner x is 12, and after the inner scope finishes, the outer x is still 6. Note that y is not shadowed, so its value changes in the outer scope.
